# 最小实测闭环执行清单（测试步骤 + 数据模板 + 拟合验收阈值）

## 1. 目标与边界

目标：在缺少高可信实测数据阶段，用最小成本建立一套可复现、可比较、可判定的参数闭环。  
范围：先覆盖常温（25°C）单体数据闭环，再扩展跨温度验证。  
当前口径：先追求相对可信和趋势正确，不承诺绝对实测级精度。

## 2. 最小测试矩阵（先做这 3 组）

### 2.1 OCV-SOC 测试（用于 OCV 曲线）
- 环境温度：25°C（298.15K）
- 流程：满电静置 -> 低倍率分段放电/充电 -> 每个 SOC 点静置取近似开路电压
- SOC 点建议：`1.0 -> 0.0`，步长 `0.05`（可先 `0.1` 降成本）
- 每点静置建议：>= 30 min（或直到 `|dV/dt|` 低于阈值）
- 产出：`ocv_points.csv`

### 2.2 0.5C 充放电全循环（用于容量、电压曲线、热趋势）
- 环境温度：25°C（298.15K）
- 放电：`1.0 -> 0.0`（0.5C）
- 充电：`0.0 -> 1.0`（0.5C + CV 到 C/20）
- 采样：1s
- 产出：`cc_cycle.csv`

### 2.3 HPPC 测试（用于 ECM 动态参数）
- 环境温度：25°C（298.15K）
- SOC 点建议：`1.0 -> 0.1`，步长 `0.1`（先做 10% 间隔最低成本版本）
- 每点协议：`1C 放电 10s -> 静置 30min -> 1C 充电 10s -> 静置 30min`
- 采样：0.1s（脉冲段至少 0.1s）
- 产出：
  - 点级时序：`hppc_point_soc_XXX.csv`
  - 点级摘要：`hppc_points.csv`

## 4. 数据质量门禁（先挡坏数据，再谈拟合）

以下任一不满足，先修数据采集，不进入拟合：
- 时间序列合法：`time_s` 单调递增、无重复、无 NaN。
- 物理边界合法：`soc` 在 `[0, 1]`，电压与温度无明显异常跳变。
- 电流符号一致：放电正、充电负，全流程不混乱。
- 重复性最低要求（同工况重复 2 次）：
  - `final_soc` 差异 <= `5e-4`
  - `min_v` 差异 <= `5e-3 V`

## 5. 拟合执行顺序（最小闭环）

### 第一步：拟合 OCV 曲线
- 输入：`ocv_points.csv`
- 输出：SOC->OCV 曲线（供 DFN/ECM 共用）
- 通过标准：
  - OCV 曲线整体单调（不出现反向大幅震荡）
  - 残差 MAE <= 10mV（常温）

### 第二步：拟合 ECM（优先 2RC）
- 输入：HPPC 点级时序 + 点级摘要
- 策略：动态优先（动态 70%，静置 30%）
- 输出：`ecm_fitted_pack_temp_2d.json`（后续可扩展温度轴）
- 通过标准（常温基线）：
  - 静置段 MAE <= 5mV
  - 动态段 MAE <= 20mV
  - 动态段 P95 <= 30mV

### 第三步：拟合热参数（最小 3 参数）
- 参数：`h`（换热系数）、`heat_capacity_scale`、`thermal_conductivity_scale`
- 输入：0.5C 循环温度轨迹 + 代表工况温升轨迹
- 通过标准（常温）：
  - 峰值温度误差 <= 2.0°C
  - 温升斜率趋势一致（不反向）

## 6. 留出验证（防过拟合，必须做）

训练集：25°C 下 OCV + 0.5C + HPPC。  
留出集（至少一组）：
- `1C` 连续充放电，或
- `-10°C / 45°C` 下任一 HPPC/循环工况。

留出通过标准（建议）：
- 端电压 MAE <= 35mV
- 关键趋势保持正确（倍率升高极化增大、低温内阻上升）
